## Setup your MonitorFlow environment

Learn how to set up MonitorFlow locally and deploy it to monitor your services.

### Installation and setup

<AccordionGroup>
  <Accordion icon="github" title="Clone the MonitorFlow repository">
    To get started with MonitorFlow, clone the repository to your local machine using the following command:
    ```bash
    git clone https://github.com/yourusername/monitorflow.git
    cd monitorflow
    ```
    
    After cloning, install the dependencies using pnpm:
    ```bash
    pnpm install
    ```
  </Accordion>
  <Accordion icon="rectangle-terminal" title="Configure your environment">
    MonitorFlow requires some basic configuration before you can start monitoring your services.
    
    1. Copy the example environment file:
    ```bash
    cp .env.example .env
    ```
    
    2. Edit the `.env` file with your specific settings:
    ```
    # Database configuration
    DATABASE_URL="your-database-connection-string"
    
    # Authentication settings
    CLERK_SECRET_KEY="your-clerk-secret-key"
    CLERK_PUBLISHABLE_KEY="your-clerk-publishable-key"
    
    # Monitoring settings
    ALERT_THRESHOLD=90
    NOTIFICATION_EMAIL="alerts@yourdomain.com"
    ```
    
    3. Set up your database using Prisma:
    ```bash
    pnpm prisma db push
    ```
  </Accordion>
  <Accordion icon="play" title="Run MonitorFlow locally">
    Start the development server to run MonitorFlow locally:
    
    ```bash
    pnpm dev
    ```
    
    This will start the Next.js development server. You can access the MonitorFlow dashboard at `http://localhost:3000`.
  </Accordion>
</AccordionGroup>
