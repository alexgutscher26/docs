Use Windsurf to help develop and maintain your StackForge application. This guide shows how to configure Windsurf for better results when working with the StackForge codebase.

## Prerequisites

- Windsurf editor installed
- StackForge repository cloned locally
- Bun package manager installed
- PostgreSQL database set up

## Project understanding

Help Windsurf understand the StackForge project structure and technology stack:

### Technology stack

StackForge uses:
- **Framework**: Next.js 14 with App Router
- **Language**: TypeScript (strict mode)
- **Styling**: Tailwind CSS with custom configurations
- **UI Libraries**: Shadcn UI, Aceternity UI, Magic UI
- **State Management**: React Context API and hooks
- **Form Handling**: React Hook Form with Zod validation
- **Authentication**: Clerk
- **Database**: PostgreSQL via Drizzle ORM
- **Package Manager**: Bun

### Directory structure

Key directories in StackForge:
- `src/app/` - Next.js app router pages
- `src/components/` - React components
- `src/db/` - Database configuration and schema
- `src/schema/` - Validation schemas
- `src/hooks/` - Custom React hooks
- `src/functions/` - Utility functions
- `src/constants/` - Application constants

### Development commands

Common commands used in StackForge development:
- `bun run dev` - Start development server
- `bun run build` - Build for production
- `bun run db:push` - Push database schema changes
- `bun run db:seed` - Seed database with sample data
- `bun run db:studio` - Open Drizzle Studio

## Configuration tips

### Environment awareness

Make Windsurf aware of the environment variables used in StackForge:
- `DATABASE_URL` - PostgreSQL connection string
- `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` - Clerk publishable key
- `CLERK_SECRET_KEY` - Clerk secret key
- `NEXT_PUBLIC_APP_NAME` - Application name
- `NEXT_PUBLIC_APP_DOMAIN` - Application domain

### Component patterns

StackForge follows specific component patterns:
1. **Atomic Design**: Components organized by complexity
2. **Container-Component Pattern**: Separation of layout and UI logic
3. **Provider Pattern**: Global state management
4. **TypeScript**: Strict typing for all components

### Database patterns

When working with the database:
1. Use Drizzle ORM for all database operations
2. Define schemas in `src/db/schema/`
3. Use `db.query.tableName.findFirst()` for lookups
4. Use `db.insert(tableName).values()` for creation

## Best practices

### Code generation

When generating code with Windsurf:
1. Follow existing code patterns and conventions
2. Use TypeScript with explicit typing
3. Implement proper error handling
4. Use Tailwind CSS for styling
5. Follow accessibility guidelines

### Component development

When creating components:
1. Place them in the appropriate directory under `src/components/`
2. Use functional components with hooks
3. Implement proper TypeScript interfaces for props
4. Use Tailwind CSS utility classes for styling
5. Add proper error handling and loading states

### API development

When creating API routes:
1. Place them in `src/app/api/`
2. Use proper HTTP methods
3. Validate input with Zod schemas
4. Implement proper error handling
5. Use serverless function best practices

### Database operations

When working with database operations:
1. Use Drizzle ORM query methods
2. Implement proper error handling
3. Use server components for data fetching when possible
4. Follow validation patterns with Zod schemas
