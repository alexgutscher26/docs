Use Claude Code to help develop and maintain your StackForge application. This guide shows how to configure Claude Code for better results when working with the StackForge codebase.

## Prerequisites

- Claude Code extension installed in your IDE
- StackForge repository cloned locally
- Bun package manager installed
- PostgreSQL database set up

## Project context

Provide Claude Code with context about the StackForge project:

### Project overview

StackForge is an AI-powered social media marketing platform built with:
- Next.js 14 with App Router
- TypeScript with strict mode enabled
- Tailwind CSS for styling
- Drizzle ORM with PostgreSQL
- Clerk for authentication
- Bun as the package manager

### Key directories

Help Claude Code understand the important directories:

- `src/app/` - Next.js app router pages
- `src/components/` - React components
- `src/db/schema/` - Database schema definitions
- `src/schema/` - Validation schemas
- `src/hooks/` - Custom React hooks
- `src/functions/` - Utility functions

### Technology patterns

Make Claude Code aware of key patterns in StackForge:

1. **Authentication**: Uses Clerk with protected routes
2. **Database**: Uses Drizzle ORM with PostgreSQL
3. **Styling**: Uses Tailwind CSS with custom configurations
4. **State Management**: Uses React Context API and hooks
5. **Form Handling**: Uses React Hook Form with Zod validation
6. **UI Libraries**: Uses Shadcn UI, Aceternity UI, Magic UI

## Best practices for Claude Code interactions

### When working with components

- Follow the existing component structure and patterns
- Use TypeScript interfaces for props
- Implement proper error handling
- Use Tailwind CSS for styling

### When working with database code

- Use Drizzle ORM query patterns
- Implement proper error handling for database operations
- Use server components for data fetching when possible
- Follow validation patterns with Zod schemas

### When working with API routes

- Place routes in the `src/app/api/` directory
- Use proper HTTP methods
- Implement proper error handling and status codes
- Validate all input data using Zod schemas

## Common StackForge tasks

### Creating a new page

1. Create a new directory in `src/app/`
2. Add a `page.tsx` file with the page content
3. Add a `layout.tsx` file if needed
4. Implement proper TypeScript typing
5. Use Tailwind CSS for styling

### Creating a new component

1. Determine the appropriate directory in `src/components/`
2. Create a new `.tsx` file with the component
3. Use TypeScript interfaces for props
4. Implement the component with proper error handling
5. Use Tailwind CSS for styling

### Extending the database schema

1. Add new models to `src/db/schema/`
2. Run `bun run db:push` to update the database
3. Create new API routes to interact with the data
4. Update TypeScript types as needed

### Adding authentication protection

1. Use Clerk's `auth()` function to check authentication
2. Redirect unauthenticated users to the sign-in page
3. Protect API routes with authentication middleware
4. Use proper error handling for authentication failures
