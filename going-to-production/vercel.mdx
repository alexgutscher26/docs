# Deploy to Vercel

This guide will help you deploy your StackForge application to Vercel - the native platform for deploying Next.js applications.

## Overview

Vercel is the recommended platform for deploying StackForge applications, offering the best performance, developer experience, and integration with Next.js features.

## Deployment Steps

### 1. Connect your Repository to Vercel

1. Go to [Vercel Dashboard](https://vercel.com/dashboard)
2. Click "New Project"
3. Import your Git repository
4. Make sure to:
   - Use Next.js as the framework preset
   - Point the root directory to your project root (not a subdirectory)

### 2. Configure Environment Variables

Set all required environment variables in the Vercel dashboard:

1. Go to your project settings in Vercel
2. Navigate to "Environment Variables"
3. Add each environment variable with the appropriate production values:

```env
# App Configuration
NEXT_PUBLIC_APP_NAME=YourAppName
NEXT_PUBLIC_APP_DOMAIN=https://yourdomain.com

# Database
DATABASE_URL=postgresql://user:password@host:port/database

# Clerk Authentication
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_live_XXXXXXXXXXXXXXXXXXXXXXXX
CLERK_SECRET_KEY=sk_live_XXXXXXXXXXXXXXXXXXXXXXXX
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/auth/signin
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/auth/signup
NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_URL=/
NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_URL=/

# Optional variables
RESEND_API_KEY=your_resend_api_key
NEXT_PUBLIC_VERCEL_ANALYTICS_ID=your_vercel_analytics_id
```

### 3. Deploy the Project

Vercel should automatically infer the project settings and deploy it correctly. The deployment process includes:

1. Installing dependencies with `bun install`
2. Building the application with `bun run build`
3. Deploying to Vercel's global CDN

### 4. Configure Custom Domain (Optional)

1. Go to your project settings in Vercel
2. Navigate to "Domains"
3. Add your custom domain
4. Follow the DNS configuration instructions

## Vercel-Specific Optimizations

StackForge includes several Vercel-specific optimizations:

### Image Optimization

Next.js automatically optimizes images when deployed to Vercel:

```tsx
// src/components/global/optimized-image.tsx
import Image from 'next/image';

export function OptimizedImage({ src, alt, width, height }: { src: string; alt: string; width: number; height: number }) {
  return (
    <Image
      src={src}
      alt={alt}
      width={width}
      height={height}
      quality={75}
      placeholder="blur"
    />
  );
}
```

### Edge Functions

Take advantage of Vercel's Edge Functions for better performance:

```ts
// next.config.mjs
export default {
  experimental: {
    runtime: 'edge',
  },
};
```

## Troubleshooting

### Common Issues

1. **Build Failures**: Check logs for missing environment variables
2. **Environment Variables Not Set**: Ensure all required variables are configured
3. **Database Connection Issues**: Verify `DATABASE_URL` format and credentials
4. **Clerk Authentication Errors**: Check Clerk keys and domain settings

### Debugging Deployment Issues

1. **Check Build Logs**: Review detailed build logs in Vercel dashboard
2. **Verify Environment Variables**: Ensure all variables are correctly set
3. **Test Locally**: Verify the build works locally with `bun run build`
4. **Check Dependencies**: Ensure all dependencies install correctly

### Vercel CLI Deployment

You can also deploy using the Vercel CLI:

```bash
# Install Vercel CLI
bun add -g vercel

# Deploy to Vercel
vercel --prod
```

## Performance Monitoring

### Vercel Analytics

Enable built-in analytics for performance monitoring:

```bash
bun add @vercel/analytics
```

```tsx
// src/components/global/providers.tsx
'use client';
import { Analytics } from '@vercel/analytics/react';

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <>
      {children}
      <Analytics />
    </>
  );
}
```

### Speed Insights

Add Vercel Speed Insights for performance monitoring:

```bash
bun add @vercel/speed-insights
```

```tsx
// src/app/layout.tsx
import { SpeedInsights } from '@vercel/speed-insights/next';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        {children}
        <SpeedInsights />
      </body>
    </html>
  );
}
```

## Advanced Configuration

### Custom Build Settings

If needed, customize build settings in your project:

```json
// vercel.json
{
  "buildCommand": "bun run build",
  "outputDirectory": ".next",
  "installCommand": "bun install",
  "framework": "nextjs"
}
```

### Environment-Specific Variables

Use Vercel's environment-specific variables:

1. **Development**: Variables with "Development" environment
2. **Preview**: Variables with "Preview" environment
3. **Production**: Variables with "Production" environment
